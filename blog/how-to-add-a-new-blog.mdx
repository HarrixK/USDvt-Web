> _This page is itself an MDX example. Follow the checklist below whenever you create a new post._

## 1 — Create the MDX file

1. In **`/blog`**, add `my-awesome-post.mdx`.  
2. Start with this front‑matter template:

   ```yaml
   ---
   title: "My Awesome Post"
   slug: "my-awesome-post"          # kebab‑case; becomes `/blog/my-awesome-post`
   date: "YYYY-MM-DD"
   author: "Your Name"
   excerpt: "One‑sentence summary for the index page."
   tags:
     - tag1
     - tag2
   ---
   ```

3. Write your content in Markdown (MDX supports inline JSX if you need it).

## 2 — Add images or diagrams (optional)

   ```
   public/
   └─ blog/
      └─ my-awesome-post/
         ├─ hero.png
         └─ diagram.svg
   ```

   Reference them in MDX with **Next.js `<Image>`** (no import statement needed):

   ```mdx
   <Image
   src="/blog/my-awesome-post/hero.png"
   alt="Descriptive alt text"
   width={800}
   height={400}
   />
   ```

## 3 — Test locally

1. Run `npm run build` or `yarn build`
2. Run `npm run dev` or `yarn dev`
3. Open http://localhost:3000/blog/my-awesome-post

## 4 — Validate the build

* Run `npm run build` or `yarn build` — it must pass.  
* Check that the post appears on **`/blog`** with its excerpt.

## 5 — Open a pull request

1. Commit only:  
   * `blog/my-awesome-post.mdx`  
   * any assets under `public/blog/my-awesome-post`  
2. Push your changes.
